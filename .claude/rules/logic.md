## 下注与结算规则 (v1.0.110)

### 选号规则
- **可选位数**：1-3位数字（如：2、23、223、331）
- **数字范围**：1-4
- **客户端限制**：
  - 最多选择3个数字位
  - 每个数字最多选2次（如22，但不能222）
  - 每局总使用数字种类不超过2种

### 赔率计算 (`calcWinMultiplier` 函数)
```javascript
// 1位数字：3倍
"2" → 3倍

// 2位异数：1倍
"23" → 1倍

// 3位同数组合：重号1.5倍，轻号0.5倍
"223" → 中2=1.5倍，中3=0.5倍
"331" → 中3=1.5倍，中1=0.5倍
```

### 客户端验证
- `digitSum14(pick) > 3` → 总位数不超过3
- `unionKindsSize(roundUsedDigits, pick) > 2` → 数字种类不超过2
- `validDigitPick(digits)` → 长度1-3

## 红包功能 (v1.0.108)

### 功能描述
- **B端房主**：在消息板底部右侧添加红包图标(hb2.png)
- **C端玩家**：在消息板底部右侧添加红包图标(hb2.png)
- **点击效果**：点击红包图标后，在消息板内显示红包图片(hb1.png)
- **自动消失**：红包图片5秒后自动隐藏

### 技术实现
1. **MessageBoard组件**：
   - 新增 `showRedPacket` 状态控制红包显示
   - 在消息板容器内绝对定位红包图片
   - 支持自定义图片显示逻辑

2. **AdminRoom组件**：
   - 添加 `showRedPacket` 状态管理
   - 添加红包按钮和点击事件处理
   - 添加5秒自动隐藏逻辑
   - 房主专属按钮（固定定位在右下角）

3. **PlayerRoom组件**：
   - 添加 `showRedPacket` 状态管理
   - 添加红包按钮和点击事件处理
   - 添加5秒自动隐藏逻辑
   - 所有玩家可用（固定定位在右下角）

### 注意事项
- 红包图片来自 public 文件夹路径 (`/hb1.png`, `/hb2.png`)
- 使用绝对定位确保红包图片始终显示在消息板右下角
- 红包图片大小限制为最大180px宽度和高度
- 避免干扰现有消息显示

## 超管创建B端账号修复 (v1.0.110)

### 问题
超管创建的B端账号无法登录

### 解决方案
- `super_admin_create_b` 显式设置 `status='active'` 和 `authorized=true`
- 添加详细日志输出用于调试：
  - 创建账号时记录 hash 长度
  - 登录时记录 status 和 authorized 值
  - 密码验证失败时记录

### 相关函数
- `hashPassword(pw)`: 使用 scrypt 生成盐值和哈希
- `verifyPassword(pw, stored)`: 验证密码哈希
